Minkube K8S 无法拉取gcr镜像以及设置proxy解决办法

2019年1月19日
| 技术

Minikube Proxy

Minikube 启动的容器拉取 gcr 或者其他墙外的镜像失败之后尝试以下操作:

  1. minikube start --docker-env http_proxy=http://192.168.1.100:1087 --docker-env https_proxy=https://192.168.1.100:1087(也可以考虑换成socks5)

  2. 启动之后可以使用 minikube ssh 手动docker pull gcr.io/****相关镜像检查代理是否成功

  3. 需要注意的是 minikube 使用的 driver 默认网络配置可能为 NAT minikube ssh之后可以ifconfig查看虚拟网卡的 ip 地址, 所以上述配置 proxy 为 192 网段, 同时 Host 的 proxy 监听必须设置为 0.0.0.0
